## 2. 调试流程

### 2.1 上电前检查

```
□ 目视检查
  □ 元器件极性正确
  □ 焊接质量良好
  □ 无短路/虚焊
  □ 连接器安装正确

□ 万用表检查
  □ 电源对地电阻 (>100Ω)
  □ I2C 总线电阻 (上拉正常)
  □ 关键信号通断

□ 供电检查
  □ 电源电压正确
  □ 极性正确
  □ 电流限制设置
```

### 2.2 上电测试

```
□ 首次上电 (限流 100mA)
  □ 电流在预期范围内
  □ 无异常发热
  □ 电源电压稳定

□ 分模块上电
  □ 数字部分 (3.3V/1.8V)
  □ 模拟部分 (AVCC)
  □ 传感器供电
```

### 2.3 功能测试

```
□ 最小系统测试
  □ MCU 运行正常
  □ 时钟正常
  □ 复位正常

□ 通信接口测试
  □ I2C 扫描设备
  □ SPI 回环测试
  □ UART 收发测试

□ 传感器测试
  □ 读取 ID 寄存器
  □ 读取数据
  □ 配置功能
```

## 3. 常见问题排查

### 3.1 电源问题

**现象**: 无法上电或电流过大

**排查步骤**:
1. 断电，测量电源对地电阻
2. 检查 LDO 输入输出电压
3. 分段断开负载定位短路
4. 检查电容是否击穿

**常见原因**:
- 电源短路 (焊接问题)
- 电容击穿
- 芯片损坏
- 极性接反

### 3.2 I2C 通信失败

**现象**: 无法读取传感器数据

**排查步骤**:
```bash
# 1. I2C 扫描
i2cdetect -y 1

# 2. 读取 ID 寄存器
i2cget -y 1 0x38 0x00

# 3. 示波器测量波形
# - 检查 SCL/SDA 波形
# - 测量上升时间
# - 检查 ACK/NACK
```

**常见原因**:
- 上拉电阻缺失 (4.7kΩ)
- 地址错误 (7 位 vs 8 位)
- 时序不匹配
- 总线电容过大
- 传感器供电异常

### 3.3 SPI 通信失败

**现象**: 数据读取错误或无响应

**排查步骤**:
1. 示波器测量 4 根信号线
2. 检查时钟极性 (CPOL) 和相位 (CPHA)
3. 验证片选时序
4. 检查 MOSI/MISO 连接

**常见原因**:
- CPOL/CPHA 配置错误
- 片选逻辑反了
- 时钟频率过高
- 走线过长导致反射

### 3.4 数据异常

**现象**: 读取数据不合理或跳变大

**排查步骤**:
1. 检查传感器供电稳定性
2. 测量通信误码率
3. 验证数据格式解析
4. 执行传感器校准
5. 检查环境干扰

**常见原因**:
- 电源噪声
- 寄存器配置错误
- 数据格式理解错误
- 传感器未校准
- EMC 问题

### 3.5 传感器无响应

**现象**: 传感器完全不工作

**排查步骤**:
1. 测量供电电压
2. 检查复位信号
3. 尝试软复位
4. 更换传感器测试

**常见原因**:
- 供电异常
- 传感器损坏
- 焊接问题
- 静电损伤

## 4. 调试技巧

### 4.1 分而治之

**原则**: 将复杂问题分解为简单问题

**方法**:
1. **硬件隔离**: 断开传感器，测试 MCU 输出
2. **软件分层**: 先测底层驱动，再测应用
3. **信号分离**: 单独测试电源、时钟、数据

### 4.2 对比法

**方法**:
1. **良品对比**: 与正常工作的板子对比测量值
2. **数据手册对比**: 与规格书参数对比
3. **历史版本对比**: 与之前版本对比

### 4.3 最小系统法

**步骤**:
1. 移除不必要的外围电路
2. 只保留核心功能
3. 逐步添加功能模块
4. 每步验证功能

### 4.4 信号注入法

**应用**:
- 注入已知信号测试响应
- 使用信号发生器
- 软件模拟输入

### 4.5 日志记录

```c
// 调试宏定义
#ifdef DEBUG
#define DBG_PRINT(fmt, ...) \
    printf("[DEBUG][%s:%d] " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
#define DBG_HEX(buf, len) \
    print_hex_dump("DEBUG: ", buf, len)
#else
#define DBG_PRINT(fmt, ...)
#define DBG_HEX(buf, len)
#endif

// 使用示例
DBG_PRINT("sensor_id = 0x%02X", id);
DBG_HEX(buffer, 16);
```

## 5. 传感器专项调试

### 5.1 IMU 调试

**常见问题**: 数据漂移

**解决方法**:
```c
// 1. 静态校准
void imu_calibrate_offset(void) {
    int32_t sum[6] = {0};
    
    // 采集 1000 次数据取平均
    for (int i = 0; i < 1000; i++) {
        imu_read_raw(data);
        for (int j = 0; j < 6; j++) {
            sum[j] += data[j];
        }
        delay_ms(1);
    }
    
    // 计算零点偏移
    for (int j = 0; j < 6; j++) {
        g_offset[j] = sum[j] / 1000;
    }
}

// 2. 温度补偿
float compensate_temperature(float raw, float temp) {
    return raw * (1.0 + g_temp_coef * (temp - 25.0));
}
```

**常见问题**: 振动敏感

**解决方法**:
- 机械减震 (硅胶垫)
- 数字滤波 (低通/滑动平均)
- 降低采样率
- 选择抗振型号 (BMI088)

### 5.2 气体传感器调试

**常见问题**: 读数不稳定

**解决方法**:
```c
// 1. 充分预热
void gas_sensor_preheat(void) {
    printf("Preheating for 24 hours...\n");
    // 首次使用需要 24 小时预热
}

// 2. 基线校准
void gas_sensor_baseline_calibrate(void) {
    // 在洁净空气中进行
    g_baseline = read_sensor();
}

// 3. 数字滤波
float moving_average(float new_value) {
    static float buffer[10];
    static int index = 0;
    
    buffer[index++] = new_value;
    if (index >= 10) index = 0;
    
    // 计算平均值
    float sum = 0;
    for (int i = 0; i < 10; i++) {
        sum += buffer[i];
    }
    return sum / 10.0;
}
```

### 5.3 温湿度传感器调试

**常见问题**: 响应慢

**解决方法**:
- 增加通风
- 避免密闭空间
- 选择响应快的型号

**常见问题**: 冷凝影响

**解决方法**:
- 避免在露点以下使用
- 增加加热功能
- 选择抗冷凝型号

## 7. 安全注意事项

### 7.1 静电防护

- 佩戴防静电手环
- 使用防静电垫
- 芯片存放于防静电袋
- 避免直接触摸引脚

### 7.2 用电安全

- 高压操作需两人
- 使用隔离变压器
- 注意电容放电
- 避免短路

### 7.3 焊接安全

- 通风良好
- 佩戴防护眼镜
- 注意烙铁温度
- 避免烫伤
