## 2. 常见问题分析

### 2.1 I2C 通信失败

**现象**: 无法读取传感器数据

**排查步骤**:
1. 检查电源电压 (万用表)
2. 检查上拉电阻 (万用表)
3. 检查 SCL/SDA 波形 (示波器)
4. 扫描 I2C 地址 (逻辑分析仪)
5. 检查时序参数 (数据手册)

**常见原因**:
- 上拉电阻缺失或阻值不对
- 地址错误 (7 位 vs 8 位)
- 时序不匹配 (时钟 stretching)
- 总线电容过大

### 2.2 SPI 通信失败

**现象**: 数据读取错误或无法通信

**排查步骤**:
1. 检查 CS 信号 (示波器)
2. 检查时钟极性/相位
3. 检查 MOSI/MISO 波形
4. 验证数据手册时序

**常见原因**:
- CPOL/CPHA 配置错误
- 片选时序不对
- 时钟频率过高
- 走线过长导致反射

### 2.3 数据异常

**现象**: 读取数据不合理或跳变

**排查步骤**:
1. 检查传感器供电稳定性
2. 检查通信误码率
3. 验证数据格式解析
4. 检查传感器配置
5. 执行校准流程

**常见原因**:
- 电源噪声
- 寄存器配置错误
- 数据格式理解错误
- 传感器未校准
- 环境干扰

### 2.4 功耗异常

**现象**: 电流消耗过大

**排查步骤**:
1. 测量总电流 (万用表)
2. 分段测量定位
3. 检查传感器工作模式
4. 检查外围电路

**常见原因**:
- 传感器未进入低功耗模式
- 上拉电阻过小
- 漏电流
- 外围电路设计问题

## 3. 调试技巧

### 3.1 分而治之

1. **硬件隔离**: 断开传感器，测试 MCU 输出
2. **软件分层**: 先测试底层驱动，再测试应用
3. **信号分离**: 单独测试电源、时钟、数据

### 3.2 对比法

1. **良品对比**: 与正常工作的板子对比
2. **数据手册对比**: 与规格书参数对比
3. **历史版本对比**: 与之前版本对比

### 3.3 最小系统法

1. 移除不必要的外围
2. 只保留核心功能
3. 逐步添加功能模块

### 3.4 日志记录

```c
#define DEBUG_PRINT(fmt, ...) \
    printf("[%s:%d] " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)

// 关键节点打点
DEBUG_PRINT("sensor_init start");
DEBUG_PRINT("sensor_id = 0x%02X", id);
DEBUG_PRINT("sensor_init end, ret = %d", ret);
```

## 4. 传感器专项调试

### 4.1 IMU 调试

**问题**: 数据漂移

**解决**:
1. 静态校准 (多位置)
2. 陀螺仪零点校准
3. 温度补偿
4. 传感器融合算法

**问题**: 振动敏感

**解决**:
1. 机械减震
2. 数字滤波
3. 降低采样率
4. 选择抗振型号

### 4.2 气体传感器调试

**问题**: 读数不稳定

**解决**:
1. 充分预热 (24 小时+)
2. 基线校准
3. 温湿度补偿
4. 数字滤波 (滑动平均)

**问题**: 交叉敏感

**解决**:
1. 多传感器融合
2. 算法补偿
3. 选择专用传感器

### 4.3 光学传感器调试

**问题**: 受环境光干扰

**解决**:
1. 物理遮光
2. 调制解调
3. 背景光消除
4. 同步检测
