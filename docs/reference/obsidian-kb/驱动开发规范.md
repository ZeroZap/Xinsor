## 1. 代码组织

### 1.1 目录结构

```
src/drivers/
├── sensor_name/
│   ├── sensor_name.h        # 头文件 (接口定义)
│   ├── sensor_name.c        # 源文件 (实现)
│   ├── sensor_name_config.h # 配置文件 (可选)
│   └── README.md            # 使用说明
├── interface/
│   ├── i2c_driver.h
│   ├── spi_driver.h
│   └── uart_driver.h
└── hal/
    ├── hal_gpio.h
    ├── hal_i2c.h
    └── hal_spi.h
```

### 1.2 文件命名

- **小写字母**: 全部使用小写
- **下划线分隔**: `sensor_name.c`
- **统一前缀**: 函数使用传感器名作为前缀

### 1.3 头文件保护

```c
#ifndef SENSOR_NAME_H
#define SENSOR_NAME_H

// 头文件内容

#endif // SENSOR_NAME_H
```

## 2. 接口规范

### 2.1 标准接口函数

每个传感器驱动必须实现以下函数:

```c
/**
 * @brief 初始化传感器
 * @param config: 配置参数 (可选)
 * @return 0=成功，负值=错误码
 */
int sensor_name_init(void *config);

/**
 * @brief 读取传感器数据
 * @param data: 数据输出指针
 * @return 0=成功，负值=错误码
 */
int sensor_name_read_data(void *data);

/**
 * @brief 配置传感器参数
 * @param param: 参数 ID
 * @param value: 参数值
 * @return 0=成功，负值=错误码
 */
int sensor_name_configure(uint8_t param, void *value);

/**
 * @brief 获取传感器状态
 * @param status: 状态输出指针
 * @return 0=成功，负值=错误码
 */
int sensor_name_get_status(void *status);

/**
 * @brief 传感器自检
 * @return 0=通过，负值=失败
 */
int sensor_name_self_test(void);

/**
 * @brief 去初始化 (释放资源)
 */
void sensor_name_deinit(void);
```

### 2.2 数据结构定义

```c
/**
 * @brief 传感器数据结构
 */
typedef struct {
    // 公共字段
    uint32_t timestamp;      // 时间戳 (ms)
    uint8_t quality;         // 数据质量 (0-100)
    int32_t status;          // 状态码
    
    // 传感器特定字段
    float temperature;       // 温度 (°C)
    float humidity;          // 湿度 (%RH)
    float pressure;          // 压力 (hPa)
    
    // 原始数据 (可选)
    uint16_t raw_temp;
    uint16_t raw_hum;
} sensor_data_t;

/**
 * @brief 传感器配置结构
 */
typedef struct {
    uint8_t i2c_addr;        // I2C 地址
    uint8_t sample_rate;     // 采样率 (Hz)
    uint8_t resolution;      // 分辨率
    uint8_t power_mode;      // 功耗模式
} sensor_config_t;
```

### 2.3 错误码定义

```c
// 通用错误码
#define SENSOR_OK              0
#define SENSOR_ERROR          -1
#define SENSOR_ERROR_TIMEOUT  -2
#define SENSOR_ERROR_COMM     -3
#define SENSOR_ERROR_PARAM    -4
#define SENSOR_ERROR_NOT_INIT -5
#define SENSOR_ERROR_BUSY     -6

// 传感器特定错误码 (从 -100 开始)
#define SENSOR_ERROR_SPECIFIC -100
```

## 3. 编码规范

### 3.1 注释规范

```c
/**
 * @brief 函数功能说明 (一句话)
 * 
 * 详细说明 (可选，多行)
 * 
 * @param param1: 参数 1 说明
 * @param param2: 参数 2 说明
 * @return 返回值说明
 * 
 * @note 注意事项 (可选)
 * @warning 警告信息 (可选)
 * @example 使用示例 (可选)
 */
int function_name(int param1, char *param2);
```

### 3.2 变量命名

```c
// 局部变量：小写 + 下划线
int sensor_id;
float temperature_value;

// 全局变量：g_ 前缀
static bool g_sensor_initialized = false;

// 常量：大写 + 下划线
#define SENSOR_MAX_VALUE 1000
#define SENSOR_I2C_ADDR  0x38

// 类型：_t 后缀
typedef struct { ... } sensor_data_t;

// 指针：p 前缀 (可选)
uint8_t *p_buffer;
```

### 3.3 函数实现

```c
// 1. 参数验证
if (data == NULL) {
    return SENSOR_ERROR_PARAM;
}

// 2. 状态检查
if (!g_sensor_initialized) {
    return SENSOR_ERROR_NOT_INIT;
}

// 3. 主要逻辑
// ...

// 4. 错误处理
if (ret != SENSOR_OK) {
    // 记录错误日志
    // 恢复状态
    return ret;
}

// 5. 返回结果
return SENSOR_OK;
```

## 4. 总线驱动规范

### 4.1 I2C 驱动

```c
/**
 * @brief I2C 写入
 * @param addr: 设备地址 (7 位)
 * @param reg: 寄存器地址
 * @param data: 数据指针
 * @param len: 数据长度
 * @return 0=成功
 */
int i2c_write(uint8_t addr, uint8_t reg, uint8_t *data, uint16_t len);

/**
 * @brief I2C 读取
 * @param addr: 设备地址 (7 位)
 * @param reg: 寄存器地址
 * @param data: 数据指针
 * @param len: 数据长度
 * @return 0=成功
 */
int i2c_read(uint8_t addr, uint8_t reg, uint8_t *data, uint16_t len);

/**
 * @brief I2C 扫描
 * @return 设备地址位图
 */
uint16_t i2c_scan(void);
```

### 4.2 SPI 驱动

```c
/**
 * @brief SPI 片选
 * @param cs: 片选引脚
 * @param select: true=选中，false=释放
 */
void spi_cs(uint8_t cs, bool select);

/**
 * @brief SPI 收发
 * @param tx: 发送数据
 * @param rx: 接收数据
 * @param len: 数据长度
 */
void spi_transfer(uint8_t *tx, uint8_t *rx, uint16_t len);
```

## 5. 测试规范

### 5.1 单元测试

```c
#include "unity.h"
#include "sensor_name.h"

void setUp(void) {
    // 测试前初始化
}

void tearDown(void) {
    // 测试后清理
}

void test_sensor_init(void) {
    int ret = sensor_name_init(NULL);
    TEST_ASSERT_EQUAL(SENSOR_OK, ret);
}

void test_sensor_read_data(void) {
    sensor_data_t data;
    int ret = sensor_name_read_data(&data);
    TEST_ASSERT_EQUAL(SENSOR_OK, ret);
    TEST_ASSERT_NOT_NULL(data.timestamp);
}

int main(void) {
    UNITY_BEGIN();
    RUN_TEST(test_sensor_init);
    RUN_TEST(test_sensor_read_data);
    return UNITY_END();
}
```

### 5.2 测试覆盖率

目标覆盖率:
- 语句覆盖：> 90%
- 分支覆盖：> 80%
- MC/DC: > 70% (安全关键)

## 7. 版本控制

### 7.1 版本号规范

遵循语义化版本 (SemVer):

```
MAJOR.MINOR.PATCH
  │     │     │
  │     │     └─ 向后兼容的错误修复
  │     └─ 向后兼容的功能增强
  └─ 不兼容的 API 修改
```

### 7.2 Git 提交规范

```
feat: 新功能
fix: 错误修复
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具
```

示例:
```bash
git commit -m "feat: 添加 AHT20 温湿度传感器驱动"
git commit -m "fix: 修复 I2C 通信超时问题"
```
